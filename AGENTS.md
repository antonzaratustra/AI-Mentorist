# AGENTS.md

## Контекст проекта

`Mentorist` — это интерактивный demo-cabinet сервиса личной стратегии. Внутри уже реализованы основные продуктовые контуры: баланс сфер, формер, цели, планнер, контакты, лента и группы. Это не маркетинговый экран, а рабочий mock-продукт на тестовых данных.

## Главное по коду

- Основной UI и логика: `components/mentorist-prototype.js`
- Тестовые данные и фабрики состояния: `components/mentorist-data.js`
- Дизайн-система и все глобальные токены: `app/globals.css`
- Документ с визуальными правилами: `docs/design-system.md`

## Что важно не сломать

- Все сценарии должны оставаться кликабельными на demo-данных
- Новые цели должны попадать в карту целей, планнер, события и связанные группы
- Состояние сохраняется в `localStorage`; не сбрасывать его без явной причины
- Базовая шрифтовая пара интерфейса: `Georgia / Avenir`

## Dev-режим

- Скрытый тест шрифтов включается через `Alt+Shift+F` или `?fontlab=1`
- Основная сборочная проверка: `npm run build`
- Для ручной проверки UI использовать локальный `npm run dev`

## Стиль работы по проекту

- Сохранять визуальный язык из `docs/design-system.md`
- Не возвращать текстовые заглушки в интерфейс
- Следить за адаптивом: rail, hero, карточки и canvas-экраны не должны разваливаться на desktop и mobile
