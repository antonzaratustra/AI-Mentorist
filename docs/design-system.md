# Mentorist Design System

## Зачем
Mentorist не должен выглядеть как набор отдельных экранов. Это единый рабочий кабинет, где личная стратегия, состояние сфер, цели, контакты и группы существуют в одном визуальном контуре.

## Принципы
1. Сначала система, потом декор. Любой цвет, свечение и градиент должны помогать читать состояние, а не спорить с контентом.
2. Четыре сферы всегда кодируются одинаково: `рост` синим, `здоровье` зеленым, `дело` желтым, `отношения` красным.
3. Темные поверхности являются базой. Светлые фоны используются только как локальная подсветка внутри темного контейнера.
4. Любая карта, облако или сеть считаются канвасом. Канвас обязан поддерживать pan/zoom и иметь явные контролы масштаба.
5. Кнопка не должна занимать больше одной смысловой строки. Длинные CTA дробятся, сокращаются или выносятся в пояснение.
6. Большие заголовки используются только для ключевого экрана или главного блока. Все остальные заголовки читаются как рабочий интерфейс, а не постер.

## Цветовая система
### База
- `--bg-950`: глубокий фон приложения.
- `--bg-900`: основной подложечный тон.
- `--surface-900`: главный контейнер.
- `--surface-800`: вторичный контейнер.
- `--surface-700`: интерактивная карточка.
- `--line-soft`: мягкая граница.
- `--line-strong`: активная граница.
- `--text-primary`: основной текст.
- `--text-secondary`: вторичный текст.
- `--text-tertiary`: поясняющий текст.

### Сферы
- `growth / blue`: интеллект, обзор, структура, связи между идеями.
- `health / green`: устойчивость, ритм, восстановление.
- `work / yellow`: результат, решения, фокус недели.
- `relationships / red`: поддержка, семья, эмоциональная связь.

### Правило подсветки
- На темной поверхности используется мягкая заливка акцента `8-18%`.
- На активном элементе используется граница или свечение `24-40%`.
- Текст на акцентной подложке всегда темный только у яркой primary-кнопки. Во всех остальных случаях текст остается светлым.

## Типографика
### Семейства
- Display: `"Avenir Next Condensed", "Arial Narrow", "Trebuchet MS", sans-serif`
- Interface: `"Avenir Next", "Trebuchet MS", sans-serif`

### Шкала
- Display XL: главная формулировка в rail или hero.
- Display L: заголовок активного модуля.
- H3/H4: названия карточек и блоков.
- Body M: основной рабочий текст.
- Body S: пояснения, подписи, вторичная аналитика.
- Label XS: капсульные подписи и статусные плашки.

### Ограничения
- Не использовать сплошной uppercase для длинных заголовков и абзацев.
- Широкий display-шрифт применяется только для коротких фраз до 2-3 строк.
- На карточках используется плотная, но не плакатная строка: line-height `1.15-1.35` для заголовков, `1.5-1.6` для текста.

## Пространство и форма
- Базовый шаг: `4px`.
- Основные интервалы: `8 / 12 / 16 / 20 / 24 / 32 / 40`.
- Радиусы: `14 / 18 / 24 / 32 / full`.
- Большие панели получают внутренний отступ не меньше `20px`.
- Списки плотных интерфейсных карточек используют вертикальный ритм `10-12px`.

## Компоненты
### Панель
- Темная многослойная подложка.
- Граница мягкая, но читаемая.
- Внутренняя иерархия строится через подзаголовок, заголовок, описание, действия.

### Кнопки
- `Primary`: яркая акцентная кнопка, используется для главного следующего действия.
- `Ghost`: темная кнопка со светлым текстом и мягкой границей.
- `Icon`: квадратная кнопка для pan/zoom и мелких управляющих действий.

### Плашки и статусы
- Статусные плашки работают только на цвете сферы или нейтральном инфо-цвете.
- Плашка не должна быть единственным носителем смысла: рядом всегда есть текст.

### Канвас
- Всегда есть подпись, что поверхность можно двигать.
- Есть zoom in, zoom out и reset.
- Мир канваса больше viewport минимум на `40%`.
- Панорамирование работает по пустому фону, клики по объектам сохраняются.

### Колесо баланса
- Четыре сегмента соответствуют сферам и всегда занимают фиксированные квадранты.
- Значение сферы выражается радиусом сегмента, а не только цифрой.
- Колесо поддерживает прямое изменение состояния через drag или шаговый control.

## Поведение
- Hover: легкий подъем и усиление границы, без резких смещений.
- Focus-visible: контрастное кольцо вокруг интерактивного элемента.
- Motion: короткие переходы `160-220ms`, без декоративной инерции.

## Адаптив
- Rail и hero не должны ломаться на `1280px`.
- На mobile rail превращается в компактную верхнюю панель с горизонтальной навигацией.
- Канвасы на мобильном сохраняют pan/zoom, но получают более низкий стартовый масштаб.

## Контраст и читаемость
- Текст на цветной подложке проверяется прежде всего на вторичных и disabled-состояниях.
- Белый фон без темного текста внутри приложения не используется.
- Если карточка окрашена сферой, описание переводится в `text-primary`, а не в приглушенный серый.
